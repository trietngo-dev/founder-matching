# 機能要件一覧

| id | functionName | description | relatedRequirement | priority |
| --- | --- | --- | --- | --- |
| FR-001 | ログイン・認証機能 | 学生および教員・管理者がシステムにログインするための認証機能。※要確認：大学のLMSや学内共通認証（SSO）連携機能は未定義のため、初期開発では独自のID/PW認証を想定。 | BR-001 | 高 |
| FR-002 | 学生アカウント登録画面・機能 | 新規にプラットフォームを利用する学生が、メールアドレス等の基本情報を用いてアカウント登録を行う画面および登録処理。※要確認：受講名簿との自動照合ルール。 | BR-001 | 高 |
| FR-003 | プロフィール登録・編集画面 | 学生が自身の専攻、学年、保有スキル（ハード・ソフト）、起業モチベーション、コミットメント時間、初期役割志向（Idea Owner / Talent）を登録・変更する画面および登録（CUD）処理。 | BR-002, BR-003, BR-004, BR-005, BR-006 | 高 |
| FR-004 | スキル自己評価入力・管理コンポーネント | プロフィール登録画面内で、スキル（技術、ビジネス、デザイン等）を選択し、複数段階の習熟度スケールで視覚的に自己評価を入力・更新するパーツ。※要確認：評価基準となる標準指標の定義。 | BR-003 | 高 |
| FR-005 | プロフィール公開・通知設定機能 | 自身のプロフィールの検索公開可否や、オファー・申請受信時のメール通知・システム内通知のON/OFFを設定する画面および更新処理。 | BR-007 | 中 |
| FR-006 | プロフィールステータス自動判定処理 | 登録されたプロフィール情報の必須項目（スキル、モチベーション等）の入力有無をシステムが検証し、「マッチング準備完了」等のステータスを自動更新するバックエンド処理。 | BR-008 | 高 |
| FR-007 | プロジェクト起案・編集画面 | Idea Ownerがプロジェクトの基本情報（名称、ビジョン、事業概要、活動計画）を入力し、新規登録・下書き保存・更新（CUD）を行う画面および処理。 | BR-009, BR-012 | 高 |
| FR-008 | プロジェクト募集ポジション設定画面・機能 | プロジェクト毎に必要な役割（エンジニア、マーケター等）および求めるスキル要件を複数追加・定義・変更・削除（CRUD）できる画面機能。 | BR-010 | 高 |
| FR-009 | プロジェクト承認ワークフロー機能 | 登録されたプロジェクトが公開される前に、教員・管理者の管理画面に承認待ちとして表示され、承認ボタン押下によって「募集中」として公開されるワークフロー機能。※要確認：教員による事前承認を必須とするかの運用ルール。 | BR-011 | 中 |
| FR-010 | プロジェクトステータス自動制御・手動変更機能 | プロジェクトの募集状態を「募集中/募集停止/成立/アーカイブ」に切り替える処理。※要確認：必要メンバー全員の結成確定時に自動で「募集停止」に遷移するロジックの要否。 | BR-013, BR-014 | 高 |
| FR-011 | AI候補メンバー推薦処理 | プロジェクトに不足しているスキルや役割を埋めるのに最適な学生を、AIモデル（過去の成功チーム構成データ等を加味）を用いてマッチング抽出し、推奨順に一覧表示する機能。※要確認：AI推薦エンジンのアルゴリズム選定と技術制約。 | BR-015 | 高 |
| FR-012 | AI候補プロジェクト推薦処理 | Talent（アイデア未所持の学生）の所有する専門スキルや興味を活かせる最適な募集プロジェクトを、AIが算出して推奨順に表示する機能。※要確認：推薦モデルのAPI連携要件。 | BR-016 | 高 |
| FR-013 | メンバー手動検索・フィルタリング画面 | 学生が、指定したスキル、専門分野、コミットメント時間、キーワード等を条件に指定し、他学生をリアルタイムで検索・リスト表示する機能（R処理）。 | BR-017 | 高 |
| FR-014 | プロジェクト手動検索・フィルタリング画面 | 学生が、求める役割、活動予定時間、業界ジャンル、キーワード等の条件を指定して、合致する募集プロジェクトを検索・リスト表示する機能（R処理）。 | BR-018 | 高 |
| FR-015 | チーム内スキル補完状況可視化チャート表示機能 | チーム内で充足しているスキルと不足しているスキル、およびオファー対象者を加えた場合のスキルカバー率の変化をレーダーチャート等の図表で動的に可視化・描画するフロントエンド機能。※要確認：使用する描画ライブラリ制限。 | BR-019 | 中 |
| FR-016 | 目標・コミットメント不一致アラート表示機能 | マッチングオファー・申請時に、双方のコミットメント時間や熱量の乖離値を算出し、乖離が大きい場合に警告（ポップアップまたはテキストアラート）を表示するバックエンド・フロントエンド機能。※要確認：警告を出す不一致判定の閾値。 | BR-020 | 高 |
| FR-017 | お気に入り（ブックマーク）登録・管理機能 | 興味のあるメンバーまたはプロジェクトをワンクリックでお気に入り登録し、マイページ内のお気に入り一覧から即座に確認・アプローチできるCRUD機能。 | BR-021 | 中 |
| FR-018 | 適合度（マッチングスコア）自動算出処理 | 検索結果や推薦リストに表示される、要件とプロフィールの適合割合（％）をリアルタイムまたは定時バッチで自動計算し、結果をDBに格納・表示するバックエンド機能。※要確認：適合スコア算出の重みパラメータ定義。 | BR-022 | 中 |
| FR-019 | ロール重複・競合検知警告表示機能 | チーム内で同一のメイン役割（例：ビジネスリーダーが2名以上など）が重複し、バランスが崩れる場合に警告インジケータを表示するフロントエンド機能。※要確認：重複禁止とするロールのビジネスルール定義。 | BR-023 | 中 |
| FR-020 | AI推薦パラメータ調整画面（管理者向け） | システム管理者が、AI推薦のスコアリング時に何を重視するか（スキル、コミットメント等）の変数や重み付けをGUI上で調整・保存できる管理機能。※要確認：詳細な調整パラメータの仕様。 | BR-024 | 低 |
| FR-021 | 加入オファー送信・管理機能 | Idea Ownerが特定のTalentに対し、オファー文面・期待する役割を添付してオファーデータを登録する機能。相手側のマイページへの通知処理を含む。 | BR-025 | 高 |
| FR-022 | プロジェクト参画希望申請・管理機能 | Talentが特定のプロジェクトに対し、志望動機・アピールスキルを記述して参画希望データを登録する機能。プロジェクトオーナーへの通知処理を含む。 | BR-026 | 高 |
| FR-023 | オファー・参画申請の諾否回答機能 | オファーまたは申請を受信した学生（またはオーナー）が、承諾または辞退（辞退理由フィードバック記述可能）を登録し、送信元へ結果を通知する更新処理。 | BR-027, BR-028 | 高 |
| FR-024 | 仮マッチング中メッセージ（チャット）機能 | オファー・参画申請が承諾された「仮マッチング」状態のペア・チームが、システム内で相互にメッセージ送受信、面談日程の調整を行えるリアルタイムメッセージ機能。 | BR-028 | 高 |
| FR-025 | 最終コミットメント合意書登録・確認機能 | 本結成前の確認フェーズとして、チーム内で合意した役割分担や活動時間をテキスト等で登録・確認する合意確認UIおよびデータベース更新機能。※要確認：法的な内容ではなく講義内の目標整合用のチェックシート構成。 | BR-029 | 中 |
| FR-026 | チーム結成確定・ロック機能 | プロジェクトオーナーが「チーム結成確定」ボタンを押下した際、チームメンバーの所属状態を固定し、他のチームへの検索や応募活動を不可にする制御機能。 | BR-030 | 高 |
| FR-027 | 結成確定に伴う他案件自動キャンセルバッチ処理 | チーム結成が確定した学生が関与していた「その他未解決のオファー・申請データ」を、バックエンドで一括して「システムによる自動辞退/取り消し」ステータスに更新する一括処理機能。 | BR-031 | 高 |
| FR-028 | 教員宛チーム結成報告承認ワークフロー | 結成が確定したチームのデータが担当教員の管理画面に送信され、教員による確認・「承認」処理によって、クラス内での正式なチームとして確定される承認フロー機能。※要確認：自動承認（教員承認不要）とする運用の有無。 | BR-032 | 中 |
| FR-029 | 受講学生一括登録・クラス割り当てCSVアップロード機能 | 教員およびシステム管理者が、講義の受講生データ（氏名、学籍番号、初期クラス割り当て）を記述したCSVファイルをシステムへ一括インポートし、アカウントを一括作成・登録する機能。※要確認：LMS等の外部システム連携がない場合の代替手段としての想定。 | BR-033 | 高 |
| FR-030 | クラス設定・マッチングスケジュール管理機能 | 教員がクラス毎のスケジュール（プロファイル登録期限日、マッチング開始日、マッチング締め切り日時）を設定・変更（CUD）する管理機能。 | BR-034 | 高 |
| FR-031 | チーム結成ステータス監視ダッシュボード（教員向け） | 教員がクラス全体のチーム結成進捗（結成完了率、仮マッチング数、未マッチング者数など）および個別プロジェクトの一覧と進捗状況をリアルタイムで視覚的に監視・検索できるダッシュボード画面機能。 | BR-035 | 高 |
| FR-032 | チーム結成結果レポートCSV出力機能 | 教員が授業での成績評価等のために、結成が承認されたチームのメンバー、プロジェクト名、登録アイデア等の詳細情報をCSV形式で一括ダウンロードする機能。※要確認：出力項目の詳細仕様。 | BR-036 | 中 |
| FR-033 | 教員用手動マッチング・チーム直接編集機能 | 期限を超過した未結成学生を、教員が管理画面から特定のチームに強制追加、または教員主導で新規チームを直接組成・確定できる緊急アサイン機能。 | BR-037 | 高 |
| FR-034 | 学期末データアーカイブ・クリーンアップバッチ処理 | 学期末または年間プログラムの終了後に、システム管理者が稼働データを「過去ログ（アーカイブ）」状態に移行させ、次の期用のアカウント・クラスデータをクリーンにする定期/手動バッチ処理機能。※要確認：データの長期保持に関するポリシー定義。 | BR-038 | 中 |
